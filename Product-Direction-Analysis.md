# 二、当前最匹配的应用场景

监测模式改变了产品性质。**它从"工具"变成了"伙伴"。**

## 场景 1：学习助手（最自然）

```
学生把手机架在桌上，摄像头对准书本/笔记

🔴 监测中：
  "你在看高等数学第三章，这章讲的是多重积分"
  "你翻到了习题页，第5题是格林公式的应用"
  "这道题划线了，需要帮忙吗？"
  
🎤 按住说话：
  "这题怎么做？"
  "先画出积分区域，然后用格林公式..."
```

**为什么匹配**：学生长时间面对固定场景，AI 持续观察有意义。翻书、划线、写笔记——这些都是"值得播报的变化"。

## 场景 2：电子维修/DIY 指导

```
摄像头对准工作台，用户双手在操作

🔴 监测中：
  "你拿起了万用表，切换到电阻档"
  "电路板上的 R12 电阻看起来烧了，颜色变深"
  "你现在在焊接，注意温度不要超过 350 度"
  
🎤 按住说话：
  "这个电容是多少微法的？"
  "丝印是 470，应该是 470μF"
```

**为什么匹配**：双手占着，不能操作手机。AI 看着工作台，主动提醒。监测模式是刚需。

## 场景 3：烹饪辅助

```
摄像头对准厨房台面

🔴 监测中：
  "你拿出了鸡蛋和面粉，准备做蛋糕吗？"
  "蛋白已经打到硬性发泡了，可以停止了"
  "烤箱预热好了，可以放进去"
  
🎤 按住说话：
  "下一步是什么？"
  "加入过筛的面粉，轻轻翻拌"
```

**为什么匹配**：手上沾着面粉，没法碰手机。AI 看着步骤，主动给指导。

## 场景 4：家庭安防/陪伴

```
摄像头对着房间/门口

🔴 监测中：
  "画面中有人进入"
  "窗户打开了，外面在下雨"
  "猫跳上了桌子"
```

**为什么匹配**：不需要交互，纯监测就够。

# 三、产品定位判断

| 定位 | 描述 | 匹配度 |
|------|------|:--:|
| **通用 AI 助手** | 什么都能聊 | ⚠️ 太泛 |
| **视觉问答工具** | 拍照→问→答 | ✅ 基本是 |
| **场景陪伴 AI** | 持续观察+适时帮助 | ✅✅ 监测模式就是 |
| **个人教练** | 指导你做某件事 | ✅✅ 学习/维修/烹饪 |
| **安防监控** | 看家+报警 | ⚠️ 非核心但可做 |

**最精准的定位**：**「场景陪伴 AI」**——不是通用助手，是在特定场景下持续观察并在合适时机提供帮助的 AI。

# 四、拓展方向

## 方向 A：垂直场景深耕（推荐优先）

提炼"场景陪伴"为产品核心，做垂直化：

| 垂直 | 特有需求 |
|------|----------|
| **学习陪读** | 识别科目、题型、解题思路，与教材对齐 |
| **维修指导** | 零件识别、步骤引导、安全提醒 |
| **烹饪助手** | 食材识别、步骤跟踪、计时提醒 |
| **艺术创作** | 构图建议、色彩分析、风格参考 |

每个方向只需要调整 System Prompt + 少量领域知识，不需要改架构。

## 方向 B：交互升级

| 能力 | 描述 | 价值 |
|------|------|------|
| **VAD 免持对话** | 检测到说话自动收音，说完自动回答 | 双手占用场景刚需 |
| **屏幕共享** | 不只是摄像头，可以分享屏幕 | 远程协助、代码 review |
| **多摄像头** | 同时看多个角度 | 维修/安防 |
| **AR 叠加** | 在摄像头画面上叠加标注 | 维修指引、学习标注 |

## 方向 C：Agent 能力

从"看和说"到"做"：

| 能力 | 示例 |
|------|------|
| **联网搜索** | "这个芯片的 datasheet 是什么" → 自动搜索+总结 |
| **调用工具** | "帮我把这个翻译成英文并发邮件" |
| **定时提醒** | "30 分钟后提醒我关火" |
| **记录笔记** | "记住这个配方" → 自动存到记忆 |

## 方向 D：协作与分享

| 能力 | 描述 |
|------|------|
| **分享视角** | 生成链接，他人可以看到你的摄像头画面 + AI 观察 |
| **多人讨论** | 多人在同一画面下各自提问 |
| **导出对话** | 把学习/维修/烹饪过程导出为教程 |

## 方向 E：平台化

| 能力 | 描述 |
|------|------|
| **Web 嵌入** | `<iframe>` 或 Web Component 嵌入第三方网站 |
| **API/SDK** | 开发者接入视觉对话能力 |
| **Provider 市场** | 第三方贡献 ASR/LLM/TTS Provider |
| **Plugins** | 第三方扩展监测规则、领域知识 |

# 五、推荐路径

```
现在（v6）
  │
  ├── Step 1: 解决部署问题 (国内可达 + 零门槛)
  │
  ├── Step 2: VAD 免持对话 (解放双手)
  │
  ├── Step 3: 垂直场景模板
  │   ├── 学习模式 (识别书本/题目)
  │   ├── 维修模式 (零件识别+安全提醒)
  │   └── 烹饪模式 (食材+步骤)
  │
  ├── Step 4: Agent 能力
  │   └── 联网搜索 + 定时提醒
  │
  └── Step 5: 分享 + 协作
```

**核心判断**：不要急着做平台、做通用 Agent。先把「场景陪伴」这个定位打磨好，选一个垂直方向（学习或维修），做出真正好用的体验。监测模式已经证明了这个方向的产品价值，下一步是让它在特定场景下变得不可替代。
